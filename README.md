# A股实时行情查询系统

专门针对A股优化的实时行情查询工具，支持股票代码和名称查询，显示实时价格、买卖量等信息。

## 趋势发现 - 完整逻辑说明（核心）

### 1. 扫描数据源

| 优先级 | 数据源 | 说明 |
|--------|--------|------|
| 主要 | 东方财富热搜榜（`hot_search_daily_snapshot` 表） | 实时更新，前700只热门股票 |
| 备用 | `HOT_STOCK_POOL` + `stock_basic_info` | 热搜榜为空时的本地股票池 |

### 2. 趋势股入选条件（中短线波段评分，6 项基础分 + 扣分 + 强制排除）

> **关注周期**：未来 ~1 个月
> **评分结构**：6 项基础分 = 100 分（满） + 扣分项（可叠加） + 强制排除项（直接剔除）

#### 2.1 维度 1：短期均线趋势（核心，40 分）

| # | 条件 | 分值 | 判定标准 |
|---|------|------|----------|
| 1 | 价格站稳 MA20 | **20 分** | 前复权价格（盘中时为实时价）持续运行在 20 日均线上方 |
| 2 | MA20 斜率向上（3 日区间） | **20 分** | 近 3 日 MA20 均值 > 前 3 日 MA20 均值（均线呈上行趋势） |

#### 2.2 维度 2：短期资金异动（活跃度，30 分）

| # | 条件 | 分值 | 判定标准 |
|---|------|------|----------|
| 3 | 近 20 日涨停次数 | **0 / 12 / 20 分** | 0 次=0 分；1 次=12 分；≥2 次=20 分 |
| 4 | 2 连板 或 连续阳线 ≥4 根 | **0 / 10 分** | 出现 2 连板及以上 **或** 连续阳线 ≥4 根，额外加 10 分 |

#### 2.3 维度 3：量能配合（短线灵魂，15 分）

| # | 条件 | 分值 | 判定标准 |
|---|------|------|----------|
| 5 | 近 20 日成交量温和放大 | **0 / 15 分** | 20 日均量 > 60 日均量 × 阈值（默认 1.0）= 15 分；缩量不得分 |

#### 2.4 维度 4：风险回撤（风控，15 分）

| # | 条件 | 分值 | 判定标准 |
|---|------|------|----------|
| 6 | 近 20 日最大回撤（分档） | **0 / 8 / 15 分** | <15% = 15 分（优秀）；15% ≤ 回撤 < 20% = 8 分（合格）；≥ 20% = 0 分（风险高） |

#### 2.5 扣分项（总分直接扣减，可叠加）

| 条件 | 扣分 |
|------|------|
| 近 5 个交易日收盘价跌破 MA20 且 3 日内无法收回 | **-15** |
| 近 20 日出现单日跌停（涨幅 ≤ -9.8%） | **-12** |
| 持续缩量阴跌（最近 5 日成交量持续 < 60 日均量 50%） | **-10** |

#### 2.6 强制排除项（无论得分多少，直接剔除）

| 条件 | 说明 | 代码位置 |
|------|------|----------|
| 近 20 日有重大利空 / 减持 / 立案调查 公告 | 通过东方财富公告接口（[services/stock.py `fetch_stock_announcements`](file:///d:/Desktop/guizhou/gupiao/server/services/stock.py)）拉取并按关键词检测 | [services/trend.py](file:///d:/Desktop/guizhou/gupiao/server/services/trend.py) `is_up_trend` exclusions |
| ST / *ST 风险警示股（短线波动无序） | 名称含 `ST`/`*ST`/`退市` 等关键词 | `has_delisting_risk` |
| 非沪深主板（北交所/新三板/科创板/创业板） | 不是 sh60/sz000/sz001 开头 | [services/stock.py `is_main_board`](file:///d:/Desktop/guizhou/gupiao/server/services/stock.py) |
| 近 20 日振幅 < 5%（完全横盘无波动） | 20 日内 (最高 - 最低) / 最低 < 5% | `TREND_RANGE_20D_FLAT_MIN` |

#### 2.7 两项门槛判断

| 门槛 | 值 | 说明 |
|------|-----|------|
| **`isUp` 门槛** | ≥ 60 分 | 认定为"趋势向上"的股票（`isUp=true`） |
| **趋势池门槛** | ≥ 85 分 | 从 `isUp=true` 的股票中筛选，最终进入趋势发现列表 |

### 3. 趋势股排除条件

以下条件满足任一即**不进入**趋势池：

| 条件 | 说明 | 代码位置 |
|------|------|----------|
| K线数据不足 | K线天数 < 60 天 | `TREND_MIN_KLINE_DAYS` |
| 强制排除（ST / 主板 / 利空 / 横盘） | 见 §2.6 | [services/trend.py `is_up_trend`](file:///d:/Desktop/guizhou/gupiao/server/services/trend.py) |
| 退市风险 | 名称含 `ST`/`*ST`/`退市` 等关键词 | [services/stock.py `has_delisting_risk`](file:///d:/Desktop/guizhou/gupiao/server/services/stock.py) |
| 非沪深主板 | 不是 sh60/sz000/sz001 开头 | [services/stock.py `is_main_board`](file:///d:/Desktop/guizhou/gupiao/server/services/stock.py) |
| 趋势评分不够 | 总分 < 85 分 | `TREND_MIN_SCORE` |

### 4. 趋势股移除逻辑

#### 4.1 自动移除（每次扫描时）

趋势扫描是**全量替换**机制：每次 `/api/scan-trend-stocks` 执行时：
1. 先 `DELETE` 数据库中当天（同 `scan_date`）的所有旧记录
2. 重新全量扫描所有股票
3. 将符合条件的 `INSERT` 入数据库

→ **不满足条件的自动消失，新符合条件的自动出现**

#### 4.2 5日/10日线跌破移除（统一入口 + 盘中实时校准）

| 机制 | 触发时机 | 说明 |
|------|----------|------|
| **统一硬过滤** | 每次趋势扫描 | `is_up_trend()` 函数第一关就拒绝，价格源为**实时价**（盘中）或**收盘价**（盘后） |
| **实时价校准** | 09:30/11:30/13:00/15:00 定时扫描 | 扫描时优先取 `priceRaw` 实时价重算 MA5/MA10，盘中跌破立即移除 |
| **定时检查** | 每天 15:05 | `check_below_ma10()` 扫描**用户股票池**（非趋势池），跌破且未收回的直接删除（兜底） |

> **设计权衡说明**：系统采用盘中实时价判断是否跌破均线，目的是尽快识别转弱信号，避免用户看到已破位的股票。
> **可能副作用**：股票盘中短暂刺破均线但收盘收复，会被临时移出趋势池，需等待下一次全量扫描（通常次日）才会重新加入。
> **替代方案**：如需更稳健的判断，可在 `config.py` 中将 `USE_INTRADAY_BREAK_CHECK` 设为 `false`，改为仅用收盘价判断（或改为"连续两个收盘价跌破"再移除）。

**优化点**：
1. **数据滞后修复**：原逻辑只用 K 线收盘价判断，无法发现盘中跌破；现增加实时价校准
2. **检查入口统一**：5日/10日线检查从两处（`is_up_trend` + `check_below_ma10`）合并为一处
3. **冗余去除**：删除 `MA60>MA120` 重复评分项，分数重新分配给其他项

### 5. 趋势评分计算流程

```
扫描池中每只股票
  ├─ 获取K线 → is_up_trend(kline)
  │   ├─ 收盘价 < MA10?  →  拒绝 ❌
  ├─ 5项条件打分    →  满分 90分
  ├─ 连续上涨加分   →  最高 +10分
  └─ 总分 ≥ 60?     →  isUp=true
├─ 获取股票名称 → 为空? → 跳过
├─ 检查退市风险 → 有风险? → 跳过
├─ 检查是否沪深主板 → 不是? → 跳过
├─ 总分 ≥ 80?   → 进入候选
└─ 按分数降序排列 → 存入数据库
```

### 6. 前端加载策略

| 场景 | 行为 |
|------|------|
| 首次打开趋势发现 | 先展示缓存数据（秒开），同时检测 `scanDate` 是否过期 |
| 数据已过期 | 后台自动触发 `/api/scan-trend-stocks` 异步扫描 |
| 数据是今天的 | 直接展示，每隔30秒（盘中）/ 5分钟（盘后）自动刷新 |
| 缓存数据为空 | 显示"正在扫描"，然后触发完整扫描 |

**盘中 / 盘后定义**：

| 状态 | 时间范围 |
|------|----------|
| **盘中** | 交易日的 **09:30-11:30** 和 **13:00-15:00** |
| **盘后** | 交易日 **15:00 之后**，以及**所有非交易日**（周末、节假日） |

**节假日处理**：非交易日不触发自动刷新；前端检测到非交易日时，直接使用缓存数据并显示"休市"。

### 9. AI 选股推荐策略（2026-06-11 优化）

针对“AI智能选股推荐”板块，系统采用了一套结合量价协同、龙头溢价和骗线过滤的综合评分算法。

#### 9.1 核心评分因子（总分 100）

| 因子 | 权重 | 优化逻辑说明 |
|------|------|--------------|
| **外盘占比** | 20% | 衡量主动买入意愿。>60% 给予高分奖励。 |
| **量比健康度** | 15% | **放量出货检测**：当量比 > 1.8 且外盘占比 < 52% 时，判定为放量滞涨/出货，评分归零。 |
| **涨幅及龙头逻辑** | 20% | **不再盲目惩罚涨幅**。区分“强势启动”与“力竭赶顶”：<br>• **3%-7%**：若配合健康量比(1.0-2.2)和强外盘(>56%)，给予 **20分满分奖励**。<br>• **涨停(>9.5%)**：若委比 > 80% 且换手适中，判定为强力封板，给予 **25分超额奖励**。 |
| **均价偏离度** | 15% | 监测分时图价格与均线的关系，偏离过远（>1.5%）视为短线超买，适当降分。 |
| **委比真实性** | 10% | **虚假封单过滤**：委比 > 99% 但外盘占比 < 60% 时，判定为虚假封单诱多，评分归零。 |
| **换手率/振幅/PE** | 20% | 辅助因子，确保个股活跃度及基本面处于安全区间。 |

#### 9.2 选股硬性过滤规则

在生成推荐名单前，系统会执行以下硬性拦截，确保“实战可操作性”：

- **基本面安全**：自动过滤名称含 `ST` 或 `退` 的风险股。
- **成交真实性**：硬性拦截“量比极高（>1.8）且外盘占比不足（<55%）”的疑似骗线股。
- **拥抱龙头**：**完全取消**对高涨幅股的硬过滤。允许涨停股入选，旨在发掘次日高开溢价或龙回头机会。

## 技术说明

- 前端：Vue 3 + Vite（`client-vue/`），Vite dev 代理到后端 `/api/*`；生产由后端挂载 `static/` 构建产物
- 后端：Python + FastAPI（`server/app/main.py`）
  - 腾讯财经API：获取股票实时行情和K线数据
  - 东方财富API：获取热搜榜数据
  - DeepSeek API：AI 智能诊断
- 数据库：MySQL，使用连接池（`dbutils.pooled_db`，定义在 `server/db/database.py`）
- 存储：通过 `trend_scan_results` 表持久化，内存缓存加速访问
- 热加载：修改 `config.py` 后自动生效，无需重启
